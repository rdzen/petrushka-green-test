2. Пользователь может изменить количество каждого товара в корзине не менее, чем до 1-го. Для удаления товара из корзины используется отдельная кнопка.
9. Если пользователь уменьшает количество товара до 0, товар удаляется из корзины.

Возможность снижения количества до 0 (с удалением товара из корзины) и отдельная кнопка для удаления товара из корзины дублируют функции друг друга.

2) …
3. В корзине может находиться не более 5 различных товаров.
4. Суммарное количество всех товаров в корзине не может превышать 20 штук.

То есть < 8шт Товара1 + 7шт Товара2 + 5шт Товара3 > можно,
а < 1шт Товара1 + 2шт Товара2 + 3шт Товара3 + 4шт Товара4 + 5шт Товара5 + 4шт Товара6 > - нет ? Чем это обусловлено? 
В ТЗ описаны ограничения корзины, но не описано, как система должна проверять эти ограничения при изменении корзины. Что проверять первым? 

3) …
5. Товары в корзине могут быть разные.

 Какую информацию несет этот пункт - для чего он?  В п.3 есть упоминание о разности товаров в корзине.

4) …
6. При попытке добавить товар, превышающий лимиты, система показывает сообщение: "Лимит корзины превышен".
Пример - пользователь делает разные действия, но система отвечает одним и тем же текстом.
Примеры действий:
* Добавляет 6-й различный товар.
* Увеличивает количество одного товара с 10 до 11.
* При 3 разных видах товара добавляет товар, из-за которого общее количество становится 21.
* Пытается добавить товар при корзине с 5 позициями и 20 единицами.

Это четыре разные причины отказа. 
Выводить одно и то же сообщение для пользователя? Каким образом отображать в логах? А если нарушены сразу лимит на количество товаров и общее количество в корзине - что считать причиной?

13. Если цена на товар изменилась в каталоге, система должна автоматически обновить ее в корзине у всех пользователей.

Эти два пункта противоречат друг другу.
